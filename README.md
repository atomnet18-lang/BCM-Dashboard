# Dashboard BCM ปี 2569

Dashboard ติดตามแผนปฏิบัติการบริหารความต่อเนื่องทางธุรกิจ (BCM) ปี 2569 — ไฟล์เดียว เปิดได้ทันที เผยแพร่ผ่าน GitHub Pages

## โครงสร้างไฟล์

```
index.html            ← Dashboard (ข้อมูลแผนฝังอยู่ในไฟล์แล้ว จาก Excel sheet 1 และ 3)
data/bcm_actual.csv   ← ผลงานจริง (export จาก SharePoint List)
```

## การเผยแพร่บน GitHub Pages

1. สร้าง repository ใหม่ แล้ว push ไฟล์ `index.html` และโฟลเดอร์ `data/` ขึ้นไป
2. ไปที่ **Settings → Pages** เลือก Source = branch `main`, folder = `/ (root)` แล้ว Save
3. เปิดได้ที่ `https://<username>.github.io/<repo>/`

## การอัปเดตผลงาน (SharePoint List → GitHub)

สร้าง List จากไฟล์ `BCM_SharePoint_List_Seed.xlsx` (1 แถวต่อรายการ แบบเดียวกับ Compliance List) แล้ว Export to CSV วางทับ `data/bcm_actual.csv`

คอลัมน์ที่ dashboard อ่าน:

| คอลัมน์ | ความหมาย |
|---|---|
| `AID` | รหัสรายการ: `n-00` = กิจกรรมหลัก, `n-01` ขึ้นไป = กิจกรรมย่อย (เช่น 4-02 = ข้อ 4.2) |
| `A_ม.ค.69` … `A_ธ.ค.69` | ผลงานสะสม % รายเดือนของกิจกรรมหลัก (0–100, เดือนไหนไม่รายงานปล่อยว่าง) |
| `Status` | ยังไม่ดำเนินการ / อยู่ระหว่างดำเนินการ / เสร็จสิ้น |
| `Note` / `Issue` | ความคืบหน้า / ปัญหาและแนวทางแก้ไข |

- กิจกรรมย่อย (AID ลงท้าย -01 ขึ้นไป) ใช้เฉพาะ Status/Note/Issue — แสดงเป็น chip ในตาราง Gantt
- dashboard ยังรองรับ CSV รูปแบบยาวเดิม (ActivityID,SubID,Month,Actual,...) ด้วย
- % รวมถ่วงน้ำหนัก = Σ (Weight × ผลงานกิจกรรม) / 100 คำนวณอัตโนมัติ
- ไฟล์ CSV ปัจจุบันเป็น **ตัวอย่าง** (ผลงาน = ตามแผน ถึง มิ.ย.) — แก้เป็นผลจริงก่อนเผยแพร่
- ถ้าไม่มีไฟล์ CSV dashboard แสดงเฉพาะแผนโดยไม่ error

## หมายเหตุการแก้ไขแผน

ข้อมูลแผน (ชื่อกิจกรรม น้ำหนัก เป้าหมายสะสมรายเดือน กิจกรรมย่อย และช่วงเวลา Gantt) ฝังอยู่ใน `index.html` ส่วน `const ACTIVITIES` — หากแผนมีการปรับ (V4) แก้ที่จุดเดียวนี้
